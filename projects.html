Credit Card Fraud Detection NN

Neural network that helps spot rare fraudulent credit card transactions.

This neural network exploration project is inspired by concepts in Vishal Rajput's Ultimate Neural Network Programming with Python. The model learns patterns from historical card transactions to identify which new ones appear risky. Because confirmed fraud is extremely rare, it assigns each transaction a risk score (0–1) rather than a binary label. A higher score signals “review more closely.” The decision threshold can be adjusted to reduce alert volume (improving precision) or maximize fraud capture (improving recall). The charts below illustrate ranking quality, calibration, and threshold trade‑offs. Goal: minimize missed fraud while keeping analyst workload manageable.

Expected vs Observed Outcomes

  • Expectation: High ROC AUC (>0.95) but modest precision at balanced (0.50) threshold due to extreme rarity.
  • Observed: ROC AUC 0.9821; Average Precision 0.6844 (strong ranking ability).
  • Precision @0.50: 0.054 (alerts mostly benign at default cutoff).
  • Recall @0.50: 0.908 (most fraud is caught early).
  • Brier Score: 0.0211 (probabilities well calibrated for decision tiers / cost modeling).
  • Takeaway: Lower the threshold to chase maximum recall only if ops capacity tolerates higher false alert load; otherwise raise threshold slightly to boost alert purity, guided by the threshold sweep chart.

Equity Quant Rating Model

Multi-factor 1–10 scoring & radar visualization across Quality, Growth, Value, Momentum, Income, Risk.

This project builds a lightweight fundamental + market factor model over equity overview data sourced via the Alpha Vantage API. Raw JSON payloads are landed in an ADLS Gen2 data lake (Raw → Staging → Curated) as partitioned Parquet. Synapse Serverless external tables (over the Parquet) power a semantic layer consumed in Power BI to render a radar chart for each symbol with the chart drillable on Sector > Industry > Symbol. A single view (db.company_overview_ranking_vw) performs type normalization, derives margins & ratios, buckets raw measures into intuitive 1–10 score bands, and applies blended weights (Quality 27.5%, Growth 20%, Value 22.5%, Momentum 15%, Income 10%, Risk 5%) to output an overall score. The design favors transparency (explicit CASE logic) over opaque ML so that each pillar remains explainable for portfolio screening or idea generation.
